# Magic-Chirp 架构说明与接口文档

本文档基于当前建议结构，说明各部分职责、潜在问题、模块通信方式，以及 4 人 2 周 MVP 阶段适合采用的接口设计。

## 1. 当前架构

```text
Magic-Chirp/
├─ server.py
├─ backend/
│  └─ src/
│     ├─ modules/
│     │  ├─ auth.py          # 注册、登录、当前用户
│     │  ├─ records.py       # 创建、编辑、删除、详情
│     │  ├─ identify.py      # AI 识别接口
│     │  └─ map_records.py   # 地图公开记录数据接口
│     ├─ schemas/
│     │  ├─ user.py
│     │  ├─ record.py
│     │  └─ identify.py
│     ├─ config/
│     │  └─ settings.py
│     └─ storage/
│        └─ files.py
├─ database/
│  ├─ models.py
│  └─ databaseControl.py
├─ frontend/
└─ uploads/
```

## 2. 这个架构是否合适


- `server.py` 只做 FastAPI 应用入口。
- `modules/` 按业务能力拆分。
- `schemas/` 统一请求和响应字段。
- `database/` 负责数据表和数据库操作。
- `storage/` 负责文件保存。
- `frontend/` 负责页面和地图交互。
- `uploads/` 保存上传文件。

需要注意的问题：

| 问题 | 说明 | 建议 |
|---|---|---|
| `databaseControl.py` 可能变大 | 所有数据库操作都写进去，后期会很乱 | MVP 可以接受，但函数要按业务命名清楚 |
| `map_records.py` 命名要明确 | 它不是展示地图，而是提供地图记录数据 | 保持这个名字，比 `map.py` 更清楚 |
| `uploads/` 不适合长期生产 | 本地文件存储迁移和备份麻烦 | 比赛 MVP 可用，正式上线迁移对象存储 |
| `schemas/` 初期可能显得多 | 小项目也可以不用拆太细 | 但前后端协作时很有价值，建议保留 |
| 前端技术未确定 | 当前只说明目录，没有说明框架 | 可以先用简单 HTML/JS，也可以用 Vue/React |

结论：这个结构适合你们当前阶段。不要再过度拆分，否则 2 周内容易把时间花在工程结构上，而不是主流程上。

## 3. 各部分职责

### 3.1 `server.py`

FastAPI 应用入口。

负责：

- 创建 `FastAPI()` 实例
- 配置跨域 CORS
- 挂载各业务模块路由
- 配置静态文件访问，例如 `/uploads`
- 启动服务时初始化数据库

不负责：

- 不直接写注册登录逻辑
- 不直接写创建记录逻辑
- 不直接处理 AI 识别逻辑
- 不直接写大量数据库操作

### 3.2 `backend/src/modules/auth.py`

用户认证模块。

负责：

- 注册
- 登录
- 获取当前用户
- 退出登录，若使用 JWT 可以前端删除 token
- 校验邮箱格式
- 生成和校验登录 token

主要依赖：

- `schemas/user.py`
- `database/databaseControl.py`
- `config/settings.py`

### 3.3 `backend/src/modules/records.py`

观鸟记录模块。

负责：

- 创建观鸟记录
- 上传图片并关联记录
- 查询记录详情
- 查询我的记录
- 编辑自己的记录
- 删除自己的记录
- 校验记录权限

主要依赖：

- `schemas/record.py`
- `storage/files.py`
- `database/databaseControl.py`
- `auth.py` 中的当前用户能力

### 3.4 `backend/src/modules/identify.py`

AI 识别模块。

负责：

- 接收图片
- 调用 AI 识别服务或模拟识别函数
- 返回 Top-N 鸟种候选
- 处理识别失败和超时

主要依赖：

- `schemas/identify.py`
- `storage/files.py`，可选
- 外部 AI API 或本地 mock 函数

### 3.5 `backend/src/modules/map_records.py`

地图记录数据模块。

负责：

- 返回所有公开记录的经纬度数据
- 支持地图摘要卡片所需字段
- 后续可扩展鸟种筛选、时间筛选、范围筛选

不负责：

- 不渲染地图
- 不调用高德地图 JS SDK

地图渲染在前端完成。

### 3.6 `backend/src/schemas/`

请求和响应字段定义。

建议使用 Pydantic。

负责：

- 定义注册请求字段
- 定义登录请求字段
- 定义记录创建请求字段
- 定义记录详情响应字段
- 定义 AI 识别响应字段

价值：

- 前后端字段统一
- 自动生成 FastAPI Swagger 文档
- 减少接口字段写错

### 3.7 `backend/src/config/settings.py`

配置文件。

负责：

- 数据库地址
- JWT 密钥
- 上传目录
- 地图 Key，若后端需要
- AI API Key
- 是否启用 mock AI

MVP 可以先写简单常量，后续再改为读取 `.env`。

### 3.8 `backend/src/storage/files.py`

文件存储模块。

负责：

- 保存上传图片
- 生成文件名
- 校验文件类型
- 限制文件大小
- 返回可访问的文件 URL

MVP 建议：

- 只支持 JPG、PNG
- 只支持单张图片
- 保存到 `uploads/records/`

### 3.9 `database/models.py`

数据模型定义。

负责：

- 定义 `User`
- 定义 `BirdRecord`
- 定义 `Attachment`
- 可选定义 `VerificationCode`

如果不用 ORM，也可以把建表 SQL 写在这里。

### 3.10 `database/databaseControl.py`

数据库操作函数。

负责：

- 初始化数据库
- 创建用户
- 根据邮箱查用户
- 创建记录
- 查询公开记录
- 查询记录详情
- 查询我的记录
- 更新记录
- 删除记录

建议函数命名：

```python
create_user(...)
get_user_by_email(...)
get_user_by_id(...)
create_record(...)
get_record_by_id(...)
list_my_records(...)
list_public_records(...)
update_record(...)
delete_record(...)
create_attachment(...)
list_record_attachments(...)
```

### 3.11 `frontend/`

前端页面。

负责：

- 登录页面
- 地图首页
- 创建记录页面
- 记录详情页面
- 我的记录页面，若时间足够
- 接入高德地图 JS API
- 调用后端 API
- 保存登录 token
- 移动端适配

### 3.12 `uploads/`

上传文件目录。

建议结构：

```text
uploads/
├─ records/
│  ├─ 1001-cover.jpg
│  └─ 1002-cover.jpg
└─ temp/
```

MVP 可以只用 `uploads/records/`。

## 4. 模块通信方式

### 4.1 前端与后端通信

前端通过 HTTP 请求调用 FastAPI 接口。

```text
frontend
   |
   | HTTP / JSON / multipart-form-data
   v
FastAPI backend
```

常见格式：

- 普通数据：`application/json`
- 图片上传：`multipart/form-data`
- 登录认证：`Authorization: Bearer <token>`

### 4.2 后端模块之间通信

后端模块通过 Python 函数调用通信。

示例：

```text
records.py
   ├─ 调用 auth.py 获取当前用户
   ├─ 调用 files.py 保存图片
   └─ 调用 databaseControl.py 保存记录
```

### 4.3 后端与数据库通信

```text
modules/*.py
   |
   | 调用数据库函数
   v
database/databaseControl.py
   |
   v
SQLite / PostgreSQL
```

### 4.4 前端与地图 SDK 通信

```text
frontend map page
   |
   | 加载高德地图 JS SDK
   v
高德地图
```

地图页面同时需要从后端获取公开记录点：

```text
frontend map page
   |
   | GET /api/map/records
   v
backend
   |
   v
database
```

## 5. 接口总览

| 方法 | 路径 | 说明 | 登录 |
|---|---|---|---|
| POST | `/api/auth/register` | 注册 | 否 |
| POST | `/api/auth/login` | 登录 | 否 |
| GET | `/api/auth/me` | 获取当前用户 | 是 |
| POST | `/api/identify` | AI 鸟种识别 | 是 |
| POST | `/api/records` | 创建观鸟记录 | 是 |
| GET | `/api/records/{record_id}` | 获取记录详情 | 视权限 |
| GET | `/api/records/mine` | 获取我的记录 | 是 |
| PUT | `/api/records/{record_id}` | 编辑记录 | 是 |
| DELETE | `/api/records/{record_id}` | 删除记录 | 是 |
| GET | `/api/map/records` | 获取地图公开记录点 | 否 |

## 6. 认证接口

### 6.1 注册

`POST /api/auth/register`

请求：

```json
{
  "email": "zhangsan@smail.nju.edu.cn",
  "password": "12345678",
  "nickname": "张三"
}
```

响应：

```json
{
  "id": 1,
  "email": "zhangsan@smail.nju.edu.cn",
  "nickname": "张三",
  "role": "user"
}
```

说明：

- MVP 可先不做邮箱验证码。
- 如果要求南大邮箱，后端校验邮箱后缀。

### 6.2 登录

`POST /api/auth/login`

请求：

```json
{
  "email": "zhangsan@smail.nju.edu.cn",
  "password": "12345678"
}
```

响应：

```json
{
  "access_token": "jwt-token-string",
  "token_type": "bearer",
  "user": {
    "id": 1,
    "email": "zhangsan@smail.nju.edu.cn",
    "nickname": "张三",
    "role": "user"
  }
}
```

### 6.3 获取当前用户

`GET /api/auth/me`

请求头：

```text
Authorization: Bearer <access_token>
```

响应：

```json
{
  "id": 1,
  "email": "zhangsan@smail.nju.edu.cn",
  "nickname": "张三",
  "role": "user"
}
```

## 7. AI 识别接口

### 7.1 上传图片识别鸟种

`POST /api/identify`

请求类型：`multipart/form-data`

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| image | file | 是 | 待识别图片 |

响应：

```json
{
  "candidates": [
    {
      "name": "珠颈斑鸠",
      "confidence": 0.86
    },
    {
      "name": "山斑鸠",
      "confidence": 0.42
    }
  ],
  "source": "mock"
}
```

说明：

- `source` 可以是 `mock`、`api`、`local_model`。
- 2 周内建议保留 mock 兜底，避免演示失败。

## 8. 记录接口

### 8.1 创建观鸟记录

`POST /api/records`

请求类型：`multipart/form-data`

字段：

| 字段 | 类型 | 必填 | 说明 |
|---|---|---|---|
| bird_name | string | 是 | 用户确认后的鸟种 |
| ai_candidates | string | 否 | JSON 字符串形式的 AI 候选结果 |
| description | string | 否 | 备注 |
| latitude | float | 是 | 纬度 |
| longitude | float | 是 | 经度 |
| location_name | string | 否 | 地点名称 |
| observed_at | string | 是 | 观鸟时间 |
| visibility | string | 是 | `public` 或 `private` |
| image | file | 否 | 图片，MVP 建议支持一张 |

响应：

```json
{
  "id": 1001,
  "bird_name": "珠颈斑鸠",
  "detail_url": "/records/1001",
  "message": "record created"
}
```

### 8.2 获取记录详情

`GET /api/records/{record_id}`

响应：

```json
{
  "id": 1001,
  "bird_name": "珠颈斑鸠",
  "ai_candidates": [
    {
      "name": "珠颈斑鸠",
      "confidence": 0.86
    }
  ],
  "description": "在鼓楼校区树边看到",
  "latitude": 32.0569,
  "longitude": 118.7792,
  "location_name": "南京大学鼓楼校区",
  "observed_at": "2026-06-01T15:30:00",
  "visibility": "public",
  "author": {
    "id": 1,
    "nickname": "张三"
  },
  "attachments": [
    {
      "file_url": "/uploads/records/1001-cover.jpg",
      "file_type": "image"
    }
  ],
  "created_at": "2026-06-01T15:31:00"
}
```

权限规则：

- 公开记录：游客可看摘要，注册用户可看详情。
- 私有记录：只有作者本人可看。
- MVP 若时间紧，可先让公开记录详情登录后可看。

## 9. 地图接口

### 9.1 获取公开地图记录点

`GET /api/map/records`

查询参数，可选：

| 参数 | 类型 | 必填 | 说明 |
|---|---|---|---|
| bird_name | string | 否 | 按鸟种筛选 |
| start_time | string | 否 | 起始时间 |
| end_time | string | 否 | 结束时间 |

MVP 可以先不实现筛选参数，但接口预留。

响应：

```json
[
  {
    "id": 1001,
    "bird_name": "珠颈斑鸠",
    "latitude": 32.0569,
    "longitude": 118.7792,
    "location_name": "南京大学鼓楼校区",
    "observed_at": "2026-06-01T15:30:00",
    "cover_image_url": "/uploads/records/1001-cover.jpg",
    "author_nickname": "张三"
  }
]
```

前端使用方式：

- 打开地图页后请求该接口。
- 遍历返回数组。
- 对每条记录创建一个地图 Marker。
- 点击 Marker 后展示摘要卡片。
- 摘要卡片点击后跳转 `/records/{id}`。

## 10. 推荐后端路由挂载方式

`server.py` 中建议类似：

```python
from fastapi import FastAPI
from fastapi.middleware.cors import CORSMiddleware
from fastapi.staticfiles import StaticFiles

from backend.src.modules import auth, identify, records, map_records

app = FastAPI(title="Magic-Chirp API")

app.add_middleware(
    CORSMiddleware,
    allow_origins=["*"],
    allow_credentials=True,
    allow_methods=["*"],
    allow_headers=["*"],
)

app.mount("/uploads", StaticFiles(directory="uploads"), name="uploads")

app.include_router(auth.router, prefix="/api/auth", tags=["auth"])
app.include_router(identify.router, prefix="/api/identify", tags=["identify"])
app.include_router(records.router, prefix="/api/records", tags=["records"])
app.include_router(map_records.router, prefix="/api/map", tags=["map"])
```

## 11. 推荐开发顺序

4 人 2 周下，建议按这个顺序：

1. 建数据库字段：`users`、`bird_records`、`attachments`
2. 完成登录接口，允许前端拿到 token
3. 完成地图公开记录接口，先返回假数据
4. 前端地图页接入高德地图并渲染假数据
5. 完成 AI 识别接口，先 mock 返回候选鸟种
6. 完成创建记录接口，支持上传一张图片
7. 前端创建记录页串起图片、AI、定位、提交
8. 完成记录详情接口和详情页
9. 联调地图新增记录后刷新展示
10. 最后补编辑、删除、我的记录
