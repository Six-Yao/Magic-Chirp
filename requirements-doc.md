# Magic-Chirp（神奇啾啾）需求文档 V0.3

> **阶段一重心：鸟种识别（AI 辅助）+ 记录。社交仅限于地图上看到他人记录。**

---

## 1. 背景与目标

### 1.1 项目背景
**校园观鸟的现状与痛点**

南京大学仙林校区及周边绿地、湿地资源丰富，记录到的鸟类超过百余种。然而校内一直缺乏统一的观鸟记录与交流平台。同学们目前多通过微信群、朋友圈、BBS 等零散渠道分享观鸟信息——图片沉在聊天记录里，位置靠口头描述，鸟种靠互相问。信息不可检索、不可积累、不可复用，观鸟行为始终停留在"个人偶遇"层面，难以形成校园范围内的集体知识。

**为什么需要一个新的平台**

市面上已有 eBird、观鸟记录中心等成熟产品，但它们面向的是专业观鸟群体，核心交互是"表单式记录 + 名录管理"，对新手的门槛偏高。一个刚对观鸟产生兴趣的学生，打开这些产品的第一反应往往是"我不认识这只鸟，我该怎么记录？"——识别门槛直接挡住了记录行为。

Magic-Chirp 的切入点在于：**用 AI 识别降低"不认识"的恐惧**，拍照即识别，识别即记录。让一个完全不懂鸟类的学生也能在 30 秒内完成一次有意义的观鸟记录。

**观鸟作为一种新的社交方式**

观鸟天然适合与当下年轻人热衷的户外活动融合。喜欢 citywalk 的人，可以在城市漫步中留意路边的鸟类；喜欢徒步、爬山的同学，可以在登山路线上记录沿途发现的鸟种。观鸟不需要额外的时间成本——它是对已有户外活动的一次"注意力升级"，让你在同样的路上看见更多。

Magic-Chirp 将观鸟定位为一种**轻量户外社交的载体**：你不需要专门为观鸟出门，只需要在散步、跑步、爬山时多看一眼树梢。你的每一次记录都会出现在校园地图上，别人的发现也会成为你下次出行的线索。地图上的标记点不只是数据，更像是其他人留下的"这里有好东西"的便签——观鸟由此从一个人的偶遇，变成一群人共同绘制的好奇心地图。

### 1.2 核心目标
为南京大学学生提供一个**AI 辅助的观鸟记录与发现平台**：
- 通过拍照进行 AI 鸟种识别，降低观鸟门槛
- 支持摄影、录像、录音等多种记录方式
- 校园及周边鸟类分布的可视化地图（所有人的公开记录汇聚一处）
- 通过地图发现校园周边的鸟类多样性

> 阶段一不包含点赞、评论、信息流等社交功能。社交仅体现在"地图上看到他人记录"这一层。

### 1.3 成功指标（MVP）
- 注册用户可在 1 分钟内完成一条观鸟记录的创建
- 地图页面可展示所有用户的公开记录，支持按鸟种、时间、地点筛选
- 无严重数据丢失或隐私安全事故

## 2. 范围与边界

### 2.1 本期（阶段一）范围
- 南京大学邮箱注册、登录
- AI 鸟种识别（上传图片 → 返回识别结果，用户确认或修正）
- 观鸟记录 CRUD（拍照/选图、录像、录音作为附件；AI 辅助识别鸟种；标记位置、时间）
- 个人记录管理（查看、编辑、删除自己的记录列表）
- 地图页：所有用户公开记录的地理分布展示（唯一的社交面）
- 记录详情页（可从地图标记点跳转）

### 2.2 本期不做什么
- 鸟种图鉴 / 百科集成
- 首页信息流、点赞、评论、分享、关注、好友（后续阶段）
- 群组、挑战、排行榜
- 私信、好友系统

### 2.3 依赖与假设
- 用户设备具有 GPS/网络定位能力
- 用户拥有南京大学邮箱（ @smail.nju.edu.cn）
- 地图底层依赖高德或 Mapbox
- AI 鸟种识别依赖预训练图像分类模型（本地部署或 API 调用）
- 服务器部署在国内，满足备案要求

## 3. 用户角色

| 角色 | 说明 |
|------|------|
| 游客 | 未登录用户，仅可浏览地图上的公开记录点（不可查看详情） |
| 注册用户 | 可创建、管理自己的观鸟记录，查看他人的记录详情（从地图进入） |
| 管理员 | 可删除违规内容、封禁用户 |

---

## 4. 功能需求

### 4.1 用户模块
| 编号 | 需求 | 优先级 |
|------|------|--------|
| U-01 | 注册：仅限南京大学邮箱（@smail.nju.edu.cn），发送验证码到邮箱完成验证 | P0 |
| U-02 | 登录 / 退出（邮箱 + 密码） | P0 |
| U-03 | 修改头像、昵称（昵称将显示在地图标记点和记录详情中） | P1 |

### 4.2 观鸟记录模块
| 编号 | 需求 | 优先级 |
|------|------|--------|
| R-01 | 创建记录：必填字段为 位置（自动定位 + 手动调整）、时间（默认当前）、鸟种名称；可选附件为 图片（最多9张）、视频（≤60s）、音频（≤60s） | P0 |
| R-02 | AI 鸟种识别：用户上传图片后，系统返回 Top-N候选鸟种及置信度，用户选择一个确认或自行输入；识别结果随记录保存 | P0 |
| R-03 | 编辑 / 删除自己的记录 | P0 |
| R-04 | 记录的可见性：公开 / 仅自己可见 | P0 |
| R-05 | 查看记录详情（含附件预览、位置地图缩略图、AI 识别结果） | P0 |
| R-06 | 搜索：按鸟种名称、地点名称模糊搜索 | P1 |

### 4.3 地图模块
| 编号 | 需求 | 优先级 |
|------|------|--------|
| M-01 | 地图展示所有公开记录的地理标记点（聚合显示，缩放后展开） | P0 |
| M-02 | 点击标记点弹出摘要卡片（鸟种、缩略图、记录时间、记录者） | P0 |
| M-03 | 点击卡片跳转至记录详情 | P0 |
| M-04 | 按鸟种、时间范围筛选地图上的记录 | P1 |
| M-05 | 默认视野锁定南京大学及周边（约 5km 范围） | P0 |

### 4.4 社交（后续阶段，本期不做）
以下功能明确不在阶段一范围内，记录于此以避免范围蔓延：
- 首页信息流（时间线）
- 点赞 / 评论 / 分享
- 关注 / 好友
- 查看他人个人主页

## 5. 非功能需求

| 编号 | 需求 | 指标 |
|------|------|------|
| N-01 | 地图页首屏加载、记录详情页加载 | < 3s（4G 网络下） |
| N-02 | 图片上传 | 单张 ≤ 10MB，自动压缩至 ≤ 2MB |
| N-03 | 视频上传 | 单条 ≤ 30MB，时长 ≤ 60s |
| N-04 | 音频上传 | 单条 ≤ 5MB，时长 ≤ 60s |
| N-05 | AI 识别响应 | ≤ 3s（从图片上传完成到返回结果） |
| N-06 | 系统并发 | 支持 ≥ 200 用户同时使用（以大赛场景为参考） |
| N-07 | 移动端适配 | 页面需适配手机浏览器（响应式），核心操作（创建记录）在移动端体验流畅 |

---

## 6. 验收标准

| 编号 | 场景 | 验收描述 |
|------|------|----------|
| AC-01 | 注册登录 | 用户使用南大邮箱注册，收到验证码后完成验证并登录，全程 ≤ 2 分钟 |
| AC-02 | AI 识别 + 创建记录 | 拍照 → AI 返回候选鸟种 → 用户确认 → 提交，全程 ≤ 30 秒 |
| AC-03 | 地图浏览 | 打开地图页，能看到至少 10 个历史标记点聚合在校园视野内，点击可展开 |
| AC-04 | 权限 | 未登录用户点击地图标记点可看摘要，但不能进入详情；点击详情时引导登录 |
| AC-05 | 编辑与删除 | 用户可编辑自己的记录，删除后有确认弹窗 |

## 7. 附录

### 7.1 术语表
| 术语 | 说明 |
|------|------|
| 观鸟记录 | 用户在特定时间、地点发现特定鸟种的记录，含可选多媒体附件 |
| 鸟种名称 | 鸟类中文常用名（如"珠颈斑鸠"），由 AI 识别建议 + 用户确认得到 |
| AI 识别 | 用户上传图片后，系统调用模型返回候选鸟种列表及置信度 |
| 标记点 | 地图上代表一条观鸟记录的图钉/气泡 |
